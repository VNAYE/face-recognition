Face Recognition Attendance System

This project is a real-time face recognition-based attendance system that uses a live camera feed to automatically mark attendance. It's designed to help automate and digitize attendance tracking in schools, colleges, or offices using face detection and recognition.

🔧 Technologies Used

  • Python – Programming language used to build the system.
  • OpenCV – A library used for image and video processing (face detection, recognition).
  • Streamlit – A Python library used to create interactive web applications easily.
  • Pandas – A library for working with structured data (CSV files in this project).
  • LBPH – Local Binary Pattern Histogram Face Recognizer (for identifying faces).
  • Pickle – Used for saving and loading Python objects like dictionaries (for names).

🗂️ Project Structure

📦 Project Folder
 ┣ 📂 TrainingImage/         ← Images of registered faces (grayscale)
 ┣ 📂 TrainingImageLabel/    ← Trained face recognition model and name mappings
 ┣ 📄 app.py                 ← Main Streamlit application
 ┣ 📄 train.py               ← Trains the model with face images
 ┣ 📄 test.py                ← Recognizes faces and marks attendance
 ┣ 📄 takeimages.py          ← Captures student face images
 ┣ 📄 students.csv           ← Stores registered student info
 ┣ 📄 attendance.csv         ← Stores marked attendance records

👨‍🎓 Step-by-Step Explanation

1. Student Registration (app.py)

In the "Register Student" section:

  • The user enters the student's name and roll number.
  • The system then uses the camera to capture multiple images of the student's face.
  • The images are saved in the TrainingImage/ folder.
  • These images are later used for training the recognition model.
  • The student’s details are saved in students.csv.

2. Image Capture (takeimages.py)

This script opens the webcam and captures grayscale images of the student’s face using OpenCV. Each image is saved with a unique filename that includes the student’s name and roll number.

3. Model Training (train.py)

  • Reads all captured images from TrainingImage/.
  • Uses OpenCV's LBPH face recognizer to train a model.
  • Saves the trained model as trainner.yml.
  • Stores a dictionary of {Id: Name} using pickle to a file called names.pkl.

LBPH is a fast and effective algorithm that compares local pixel patterns in faces. It works well even in different lighting conditions.

4. Face Recognition and Attendance (test.py)

  • Opens the webcam and starts detecting faces in real-time.
  • For each detected face, it tries to recognize who it is using the trained model.
  • If recognized (confidence < 50), it marks the attendance using the mark_attendance() function.
  • If not recognized, it shows a message: "You're not registered".
  • Attendance is saved in attendance.csv with name, roll number, date, and present status.

5. Viewing Records (app.py)

  • Using Streamlit's interface, you can view both students.csv and attendance.csv.
  • Duplicates are removed using Pandas.
  • It displays this data in a readable table format on the webpage.

🧠 What is LBPH Face Recognizer?

LBPH (Local Binary Patterns Histograms) is an algorithm used for facial recognition:

  • It converts each pixel to a binary value by comparing it to its surrounding neighbors.
  • This binary pattern is then used to create a histogram.
  • These histograms are compared to known faces to recognize a person.
